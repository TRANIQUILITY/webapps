Model Context Protocol (MCP)이란?

MCP는 Gemini AI 모델이 터미널 환경에서 파일 시스템 접근, 코드 실행 등 외부 도구와 상호작용하여 능동적인 AI 에이전트로 작동하게 하는 개방형 프로토콜(규약)입니다.

통신 구조

Gemini CLI가 외부 도구를 사용하는 과정은 여러 구성 요소 간의 정교한 상호작용을 통해 이루어집니다.

① 클라이언트 (Gemini CLI)

사용자 PC에서 실행되며 모든 통신의 시작점이자 전체 흐름을 조율합니다.

② Gemini 모델 API

사용자 요청을 해석하여 어떤 '도구'가 필요한지 결정하고 실행을 지시하는 두뇌 역할을 합니다.

③ MCP 서버

클라이언트의 요청에 따라 실제 외부 API를 호출하고 결과를 반환하는 중간 실행자입니다.

④ 외부 도구 API

Google Calendar, Gmail API 등 실제 기능이 구현된 최종 목적지입니다.

사전 설정 가이드

1️⃣

Node.js 설치

Gemini CLI 실행을 위해 Node.js v20 이상 버전 설치가 필요합니다.

2️⃣

Gemini CLI 설치

터미널에 npm install -g @google/gemini-cli 명령어로 CLI를 설치합니다.

3️⃣

최초 실행 및 인증

터미널에 gemini를 입력 후 Google 계정으로 로그인하여 인증을 완료합니다.

1인 법인을 위한 오피스 업무 자동화

Gemini CLI의 진정한 힘은 사용자 정의 MCP 서버를 통해 확장할 때 나타나며, 외부 서비스 API와 연동하여 Gemini에 새로운 업무 처리 능력을 부여할 수 있습니다.

MCP 마켓플레이스 활용하기

모든 MCP 서버를 직접 개발할 필요는 없으며, mcpmarket.com이나 smithery.ai와 같은 마켓플레이스에서 공유된 서버를 찾아 CLI의 기능을 손쉽게 확장할 수 있습니다.

마켓플레이스 서버 연동 예제 (Notion)

  1. 1. 서버 탐색 및 설정 정보 확인

    mcpmarket.com에서 'Notion' MCP 서버를 찾은 후, 연동에 필요한 설정 정보와 환경 변수를 확인합니다.

  2. 2. settings.json 파일에 서버 추가

    홈 디렉토리의 .gemini/settings.json 파일에 확인한 설정 정보를 추가합니다.

    { "mcpServers": { "notion": { "command": "npx", "args": ["-y", "@mcp-market/notion"], "env": { "NOTION_API_KEY": "YOUR_NOTION_API_KEY" } } } }
  3. 3. Gemini CLI에서 새로운 도구 사용

    터미널에서 '--mcp' 옵션으로 'notion' 서버를 지정하여 Notion 관련 작업을 직접 요청할 수 있습니다.

    gemini --mcp notion -p "내 Notion '업무일지'에 오늘 날짜 페이지 만들고 '주간 회의 참석' 내용을 추가해줘."
    Notion MCP 서버를 실행합니다...\n'업무일지' 데이터베이스에 새로운 페이지를 생성하고 내용을 추가했습니다.

실전 프로젝트: 나만의 데일리 브리핑 봇

여러 도구를 조합하여, gemini -p "오늘의 브리핑 해줘" 명령어로 오늘의 주요 정보를 요약하는 나만의 에이전트를 만들어 봅니다.

프로젝트 목표 및 사용 도구

오늘의 뉴스, 첫 캘린더 일정, 로컬 할 일 목록을 종합하여 하나의 브리핑으로 요약합니다.

webpagecustom_calendar_mcpfilesystem
  1. 최종 단계: 모든 정보를 종합하여 브리핑 요청

    Gemini의 자연어 처리 능력을 활용하여, 여러 도구의 작업을 하나의 프롬프트로 종합하여 최종 브리핑을 생성합니다.

    gemini --mcp calendar -c ~/todo.txt -p "https://news.site.com 뉴스를 요약하고, 오늘 첫 일정과 내 todo.txt의 할 일을 종합해서 데일리 브리핑을 만들어줘."
    알겠습니다. 각 도구를 사용하여 정보를 수집하고 종합합니다...

문제 해결 (FAQ)

"gemini: command not found" 오류가 발생합니다.

Node.js와 npm의 PATH 환경변수가 제대로 설정되지 않은 경우입니다. 터미널을 재시작하거나, NVM을 사용 중이라면 nvm use 명령어로 버전을 다시 활성화해 보세요.

Google 계정 인증이 반복적으로 실패합니다.

인증 토큰이 만료되었을 수 있습니다. 홈 디렉토리의 .gemini 폴더 내 인증 관련 파일(예: google_accounts.json)을 삭제한 후 gemini를 다시 실행하여 재인증을 시도해 보세요.

사용자 정의 MCP 서버에 연결할 수 없습니다.

먼저 MCP 서버가 로컬에서 정상적으로 실행 중인지 확인하고, .gemini/settings.json 파일에 입력한 서버의 URL과 포트 번호가 올바른지 다시 한번 확인해야 합니다.
