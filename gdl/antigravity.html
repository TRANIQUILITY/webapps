Getting Started

Antigravity 기본 설정

Vibe Coding을 시작하기 위한 4가지 필수 준비 단계입니다.

1

IDE 설치

Google 공식 사이트에서 Antigravity IDE (VS Code 기반)를 다운로드하여 설치합니다.

brew install antigravity
2

Google 계정 연동

Gemini 모델 사용을 위해 Google Cloud 또는 개인 계정으로 로그인합니다.

Settings > Account > Sign in
3

모델 선택

프로젝트 성격에 맞는 모델을 선택합니다. 최신 Antigravity에서는 'Gemini 3.0 Pro'가 기본값으로 권장됩니다.

Chat Model: Gemini 3.0 Pro
4

Agent 권한 부여

터미널 실행 및 파일 생성을 위해 에이전트에게 'Auto-Approve' 또는 'Ask' 권한을 설정합니다.

Agent Mode: Delegated
Paradigm Shift

Director vs. Builder

Vibe Coding은 단순한 코드 생성을 넘어섭니다. 당신은 프로젝트의 방향(Vibe)을 결정하는 감독(Human)이고, Google Antigravity 내의 Gemini는 실제 작업을 수행하는 에이전트(AI)입니다.

  • ✓ Human: 기획, 검수, 의사결정 (Vibe 설정)
  • ✓ AI (Gemini): 코딩, 터미널 명령, 파일 생성, 배포

노력의 변화 (Traditional vs Vibe)

* 개념적 데이터 시각화

The Vibe Loop: 핵심 프로세스

Antigravity 개발의 3단계 순환 구조를 클릭하여 확인하세요.

1

기획하기 (Planning)

Agent에게 무엇을 만들지 설명하고 청사진을 그립니다.

2

맡기기 (Delegating)

코딩뿐만 아니라 터미널 명령, 파일 생성까지 위임합니다.

3

다듬기 (Refining)

결과물을 미리 보고 대화하듯 디테일을 수정합니다.

기획하기 (Planning with Agent)

Antigravity의 사이드바(Chat)를 열고 Gemini에게 기획을 요청하는 단계입니다. 단순히 "만들어줘"가 아니라, 프로젝트의 성격과 요구사항을 정의하는 과정입니다.

💡 Pro Tip

"이 프로젝트를 위한 `PRD.md` (제품 요구사항 문서)를 작성해 줘"라고 요청하면, 에이전트가 프로젝트의 청사진을 그립니다.

Step-by-Step Guide

Project Deep Dive: Make 10 게임 만들기

Web 기술로 핵심 로직을 만들고, Mobile과 Desktop으로 확장하는 Vibe Coding 전략입니다.

Step 1: 핵심 로직 구현 (Web)

Phaser.js를 사용하여 브라우저에서 실행 가능한 프로토타입 생성

Prototype

Director's Prompt

"HTML5 Canvas 게임 엔진인 Phaser.js를 사용하여 'Make 10' 퍼즐 게임을 만들어줘.
1. 10x10 그리드에 랜덤한 숫자(1~9) 과일 아이콘 배치.
2. 마우스로 드래그하여 선택한 과일 숫자의 합이 10이 되면 사라지고 점수 획득.
3. 데스크톱과 모바일 화면 모두에 꽉 차게 반응형(Scale Manager) 설정."

Step 2: 모드 확장 (Refactoring)

기본 기능에 다양한 게임 모드(Time Attack, Zen) 추가

Expansion

Director's Prompt

"기본 게임 로직을 'ClassicMode' 클래스로 분리해줘. 그리고 다음 모드를 추가하고 싶어.
1. 'TimeAttack': 60초 제한 시간이 있고, 블록을 깰 때마다 시간이 조금씩 추가됨.
2. 'ZenMode': 시간 제한 없이 무한으로 즐기는 모드.
메인 메뉴 씬(Scene)을 만들어서 모드를 선택할 수 있게 UI를 구성해 줘."

Step 3: Mobile 패키징 (Capacitor)

iOS/Android 앱으로 변환 및 터치 최적화

Mobile

Director's Prompt

"이 웹 프로젝트에 Capacitor를 추가해서 Android와 iOS 앱으로 빌드할 수 있게 설정해 줘.
1. `npx cap init` 실행 및 설정.
2. 모바일 상태바(Status Bar)를 숨겨서 전체 화면 게임처럼 보이게 설정.
3. 터치 입력 반응 속도를 높이기 위해 CSS `touch-action: none` 적용."

Step 4: Desktop 패키징 (Electron)

MacOS/Windows 실행 파일 생성

Desktop

Director's Prompt

"Electron을 사용하여 이 게임을 데스크톱 앱(exe, dmg)으로 패키징하고 싶어.
1. Electron 메인 프로세스(`main.js`)를 작성해서 게임 `index.html`을 로드해 줘.
2. 창 크기를 800x600으로 고정하고, 메뉴바는 숨겨줘.
3. `electron-builder`를 사용하여 배포용 파일을 빌드하는 스크립트를 `package.json`에 추가해 줘."

💡 Pro Tips

@Mentions 활용

Artifacts 관리
