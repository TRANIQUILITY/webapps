운영 규칙

AI를 단순한 조수가 아닌, 각 분야의 전문성을 가진 '가상 팀원'으로 활용하여
복잡한 의사결정과 실행을 자동화하는 혁신적인 워크플로우를 소개합니다.

워크플로우 탐험 시작하기

핵심 컨셉: AI로 정의한 Role 기반으로 운영하는 기업

AI에게 각기 다른 전문가 페르소나를 부여하여 다양한 관점에서 균형 잡힌 인사이트를 얻습니다.

목표: 1인 기업이 AI를 통해 10명 이상의 전문가 팀처럼 일할 수 있음을 증명합니다.

역할 (Persona) 호출 기호 핵심 키워드

워크플로우 체험: AI 경영 회의

아래 안건에 대해 각 페르소나 버튼을 클릭하여, AI가 생성하는 다양한 관점의 의견을 직접 확인해보세요.

회의 안건 (단일 진실 공급원 - SSoT)

"개인화 투자 뉴스레터 서비스에 'AI 포트폴리오 분석' 기능을 추가한다."

목표: 사용자의 실제 포트폴리오를 분석하여 가장 관련성 높은 뉴스와 리포트를 매칭, 유료 전환율 5% 증가.
제약: 1인 개발, 3개월 내 MVP 출시 목표.

페르소나를 선택하여 의견을 확인하세요.

회의 결과 → 실행

다양한 의견을 종합하여, 다음과 같이 구체적인 실행 계획(Jira Task)으로 전환합니다.

TASK-101 To Do

'AI 포트폴리오 분석' 기능 MVP 개발

설명: 외부 증권사 API 연동 없이, 사용자가 수동 입력한 종목 기반으로 뉴스 키워드 매칭 기능 우선 개발. 1개월 내 프로토타입 완료 목표.

담당자: 👑

시스템 구성: 하이브리드 자동화

"생각은 내 PC 안에서, 행동은 클라우드에서"
로컬 AI의 보안/비용 효율성과 SaaS의 협업 기능을 결합합니다.

🧠 로컬 업무 (Local Server)

  • AI 두뇌 (Ollama/LM Studio):
  • 멀티-페르소나 시뮬레이션
  • 민감 데이터 분석 및 요약
  • 콘텐츠/코드 초안 생성
  • 지휘 본부 (n8n):
  • 전체 워크플로우 실행 및 제어
  • 데이터 변환 및 로직 처리

🔄 연동 업무 (n8n)

  • 로컬 LLM의 분석 결과 → Jira Task 내용으로 변환 후 생성
  • Gmail의 특정 이메일 수신 → 트리거로 사용하여 로컬 LLM에 분석 요청
  • 로컬에서 생성된 콘텐츠 초안 → Google Docs에 회의록으로 자동 저장
  • 주요 업무 완료 시 → Slack으로 결과 및 링크 자동 보고

☁️ 클라우드 업무 (SaaS)

  • 프로젝트 관리 (Jira):
  • Task 생성, 상태 업데이트, 추적
  • 지식 관리 (Confluence/Notion):
  • 최종 문서(회의록, 기획서) 저장/공유
  • 커뮤니케이션 (Slack/Gmail):
  • 결과 보고, 알림, 외부 소통
