행동 분석 AI 알고리즘 구현 방법과 시스템

핵심 청구항 (System)

1. 사용자 행위 데이터 수집부

사용자의 마우스 움직임, 클릭, 스크롤, 페이지 체류 시간, 입력 패턴 등 정형/비정형의 모든 상호작용 데이터를 실시간으로 수집하여 원시 데이터 스트림을 생성합니다.

2. 서비스 온톨로지 매핑부

수집된 원시 데이터에 '맥락'을 부여합니다. "어떤 버튼을 클릭했는가"를 넘어 "가격 비교 페이지에서 구매 버튼을 클릭했다" 와 같이, 서비스의 기능적/정보적 구조(온톨로지)와 행위를 연결합니다.

3. 다차원 의도 분석부

단순 행동이 아닌 '의도'를 추론합니다. 여러 행동을 묶어 '상품 비교' 클러스터로 분석하고, 반복적 행동 패턴에서 '구매 망설임'이라는 복합 의도를 추론하며, 의미 없는 행동은 '의도 없음'으로 판단합니다.

4. 선제적 서비스 제공부

추론된 의도를 바탕으로 가장 효과적인 개입을 수행합니다. '구매 망설임' 의도에는 할인 쿠폰을, '정보 부족' 의도에는 상세 스펙 비교표를 제시하는 등 개인화된 서비스를 능동적으로 제공합니다.

차별화 포인트

'언어'가 아닌 '행동' 기반 의도 추론

사용자가 채팅창에 "도와주세요"라고 입력하기 전, 이미 수십 번의 마우스 클릭과 페이지 이동으로 표현된 암묵적인 니즈를 먼저 포착합니다.

'결과'가 아닌 '과정'상의 의도 분석

"이 사용자는 80% 확률로 이탈할 것이다"라는 결과 예측을 넘어, "가격 정보와 리뷰 페이지를 5번 이상 왕복했으므로, 가격 대비 성능에 대한 확신이 부족한 상태"라는 과정상의 의도를 분석합니다.

'정적 규칙'을 넘어선 '지능적' 개입

"장바구니에 30분 이상 있으면 쿠폰 제공"이라는 고정된 규칙이 아닌, 사용자의 행동 맥락에 맞춰 "두 상품의 스펙 차이가 고민이신가요? 핵심 비교 자료를 보여드릴게요"와 같이 동적이고 지능적인 상호작용을 수행합니다.

범용적 '통합 플랫폼' 아키텍처

이커머스 챗봇, 온라인 교육 튜터, SaaS 온보딩 가이드 등 AI 에이전트의 종류와 무관하게, 행위 분석과 의도 추론이라는 핵심 로직을 모듈처럼 적용할 수 있는 높은 확장성을 가집니다.

산업별 적용 사례 (Use Cases)

1. 이커머스: 구매 전환율 극대화

사용자가 A노트북(고성능, 고가)과 B노트북(가성비) 페이지를 3회 이상 왕복하고, 두 제품의 비교 리뷰 블로그로 나갔다가 다시 돌아오는 행동 패턴을 보입니다. 시스템은 이 행동 클러스터를 '최종 구매 직전, 성능과 가격 사이에서의 심층적 고민' 의도로 추론합니다. 단순 쿠폰 제안이 아닌, "두 제품의 실제 게임 프레임 비교 영상과 렌더링 속도 벤치마크 데이터를 보여드릴까요?"라는 메시지를 띄워, 사용자의 핵심적인 고민(가격 차이만큼의 성능 차이가 있는가?)을 정확히 해결해 줌으로써 구매 결정을 촉진합니다.

2. 온라인 교육 플랫폼: 학습 이탈 방지

한 수강생이 '미적분' 강의의 특정 구간(3분 15초~4분 30초)을 5번 이상 반복 재생하고, 관련 퀴즈에서 오답률 80%를 기록한 뒤 '수강 포기' 버튼 근처에서 마우스를 10초 이상 머무릅니다. 시스템은 이를 단순한 '개념 이해 부족'을 넘어 '학습 좌절로 인한 이탈 임박' 의도로 판단합니다. 즉시 "해당 개념을 초등학생도 이해할 수 있게 설명한 5분 요약 강의를 먼저 보시겠어요? 잠시 쉬어가는 것도 좋은 방법입니다." 와 같은 감성적 접근과 함께 대안적 학습 경로를 제시하여 수강생의 학습 의지를 회복시킵니다.

3. SaaS/B2B 소프트웨어: 고객 온보딩 및 리텐션 강화

신규 가입자가 서비스의 핵심 기능인 '자동 리포트 생성' 페이지에 진입했으나, 데이터 소스 연결 단계에서 3분 이상 머무르며 여러 입력 필드를 클릭했다 지우기를 반복합니다. 시스템은 이를 '초기 설정의 어려움으로 인한 기능 도입 실패' 의도로 정의합니다. 이때 "혹시 CSV 파일 업로드에 어려움을 겪고 계신가요? 샘플 템플릿을 다운로드하시거나, 1분 동영상 가이드를 확인해 보세요." 라는 툴팁을 띄워 사용자가 첫 번째 성공 경험(A-ha Moment)을 빠르게 달성하도록 유도하고 장기 고객으로 전환될 가능성을 높입니다.

4. 금융 서비스: 비대면 상품 가입 유도

사용자가 A, B, C 신용대출 상품의 상세 페이지를 모두 조회하고, 특히 '중도상환수수료'와 '필요 서류' 항목을 여러 번 클릭하며 스크롤을 내립니다. 이후 가입 절차를 시작했다가 '공인인증서 로그인' 단계에서 이탈합니다. 시스템은 이를 '상품 조건은 만족했으나, 가입 절차의 복잡성 및 비대면 인증의 불편함' 의도로 분석합니다. 다음 날 사용자에게 "어제 보셨던 대출 상품, 이제 간편인증(PIN, 지문)만으로 3분 내 신청이 가능합니다." 라는 알림을 보내, 가장 큰 허들을 제거해 줌으로써 가입을 완료시킵니다.

5. 디지털 헬스케어: 환자 여정 최적화

한 사용자가 웹사이트에서 '허리 디스크' 관련 증상 정보를 2회 검색하고, '비수술 치료' 전문 의사 3명의 프로필을 조회한 뒤, '온라인 상담' 게시판의 다른 사용자 질문들만 10분 이상 읽어봅니다. 시스템은 이 일련의 행동을 '진료는 받고 싶지만, 자신의 상태를 직접 질문하는 것에 대한 두려움 또는 막연함' 의도로 해석합니다. 이때 "다른 환자분들이 자주 묻는 질문 TOP 5와 전문의의 답변을 정리해 드릴까요? 익명으로 간단한 자가진단 테스트도 가능합니다." 라고 선제적으로 제안하여, 사용자의 심리적 부담을 덜어주고 실제 상담 예약으로 이어지는 다리를 놓아줍니다.

기본 동작 흐름 (Input → Processing → Output)

  1. Input (데이터 수집): 사용자가 웹사이트에서 마우스를 움직이거나 클릭할 때마다, 프론트엔드의 경량 JS 스크립트가 해당 이벤트(좌표, 타겟 엘리먼트, 타임스탬프 등)를 캡처하여 Kafka와 같은 메시지 큐로 즉시 전송합니다.
  2. Processing (실시간 처리 및 분석):
    • Flink 스트림 프로세서가 Kafka의 이벤트 스트림을 실시간으로 소비(consume)합니다.
    • MongoDB에 저장된 서비스 온톨로지(예: '이 버튼은 구매 확정 버튼이다')와 사용자 프로필을 조회하여, 단순 이벤트를 '구매 확정 페이지에서 20초간 머무름'과 같은 의미 있는 '맥락적 행동'으로 변환합니다.
    • 최근 N초간의 행동 시퀀스를 TensorFlow 서빙 API로 전달합니다. 학습된 의도 추론 모델(LSTM, Transformer 등)이 이 시퀀스를 분석하여 '구매 망설임 (85%)', '정보 부족 (10%)' 등 확률적인 의도 벡터를 반환합니다.
  3. Output (선제적 서비스 제공): 추론된 의도 중 가장 확률이 높은 '구매 망설임'에 대해, 시스템은 사전에 정의된 액션 API를 호출합니다. 이 API는 사용자에게 "결제 시 사용 가능한 10% 할인 쿠폰이 있습니다." 와 같은 맞춤형 메시지를 프론트엔드로 전송하여 화면에 표시하도록 합니다.

반복 학습 및 최적화 (Feedback Loop)

시스템의 개입 결과(사용자가 쿠폰을 사용했는지, 제안을 무시했는지 등) 역시 새로운 이벤트 데이터로 수집됩니다. 이 피드백 데이터는 의도 추론 모델을 주기적으로 재학습시키는 데 사용됩니다. 이를 통해 시스템은 '어떤 상황에서 어떤 개입이 가장 효과적인가'를 스스로 학습하여, 시간이 지남에 따라 점점 더 정확하고 정교한 상호작용을 수행하게 됩니다.
